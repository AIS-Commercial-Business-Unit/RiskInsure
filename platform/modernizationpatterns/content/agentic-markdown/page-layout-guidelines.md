# Page layout guidelines

## Landing page
1. Hero with headline, summary, and north-star callout
2. Section anchor nav
3. Operating model visual band
4. Pillars grid
5. Lifecycle grid
6. Offerings strip
7. Use-case strip
8. CTA band

## Pillar pages
1. Hero with summary
2. Section anchor nav
3. Why it matters
4. Key decisions
5. Model components
6. Human roles and agent roles
7. Controls and artifacts
8. Metrics
9. Related lifecycle phases

## Lifecycle pages
1. Hero with summary
2. Section anchor nav
3. Why it matters
4. Major activities
5. Inputs and outputs
6. Roles
7. Controls
8. Metrics
9. Related pillars

## Design behavior
- Use generous whitespace
- Make each section visually distinct
- Keep text blocks short
- Prefer concise card grids to long lists where possible
